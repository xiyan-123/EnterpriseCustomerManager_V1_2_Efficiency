企业客户管家 V1.2 业务效率增强版

一、核心规则保持不变
1. 以企业为核心，企业名称为唯一识别标志。
2. 分组不参与唯一识别；两次导入分组不同但企业名称相同，仍合并为同一家企业。
3. 联系人、电话、备注可多次导入并不断积累。
4. 每次导入都可使用“联系人1、电话号码1、备注1”；不会因表头编号相同覆盖旧内容。
5. 新电话号码追加，重复电话号码跳过；新备注追加，重复备注跳过。
6. 客户状态默认未设置，只能在 App 内手动设置为“关注、跟进、重点”。
7. 导入手机通讯录名称为“公司名称-联系人姓名”。
8. 只删除由本 App 导入的手机通讯录联系人。

二、V1.2 新增功能
1. 通话记录匹配：主动读取最近通话记录，显示手机号、通话类型、时间、时长、系统联系人名称、App 匹配企业等信息。
2. 未入库号码处理：可选择加入已有企业、新建企业、忽略、标记无效。
3. 加入已有企业流程：点击手机号 → 搜索企业 → 选择企业 → 填写联系人/备注 → 确认添加。
4. 粘贴识别快速录入：支持粘贴一段客户信息，自动识别公司、联系人、电话、地址、备注，预览确认后入库。
5. 号码星级：每个号码支持未标记、1星、2星、3星；企业详情和通讯录页显示星级，导入通讯录时高星级优先。
6. Excel 星级字段：支持“星级1、星级2……”或“号码星级1”等表头；导出宽表也带星级。
7. 搜索结果批量入组：可将当前搜索/筛选结果批量加入已有分组或新建分组。
8. 分组导入通讯录：每个分组支持按企业序号区间导入手机通讯录。
9. 分组详情增强：查看企业数、联系人数、已导入数、星级号码统计。
10. 操作日志：记录快速录入、通话记录、批量分组、号码星级等关键操作。

三、权限说明
1. 读取通话记录需要 READ_CALL_LOG 权限；用户不授权时，通话记录匹配不可用，但其他功能不受影响。
2. 通讯录导入和同步删除需要 READ_CONTACTS / WRITE_CONTACTS 权限。
3. 数据均保存在手机本地，不上传服务器。

四、建议使用流程
1. 大批量客户：通过 Excel 宽表导入。
2. 零散客户：通过“快速录入/粘贴识别”补充。
3. 每日电话：通过“通话记录匹配”统一处理未入库号码。
4. 客户开发：搜索企业 → 批量加入分组 → 分组按序号导入通讯录。
5. 号码重要程度：在企业详情页给核心号码标记 1-3 星。
